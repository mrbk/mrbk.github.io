Setting up python for scientific computing (especially for astronomy / astrophysics) on Mac OS X


Python, its numerical computing modules numpy and scipy, and its plotting module matplotlib are great tools for the numerical tasks that scientists need to do on a daily basis. Here is how I installed python on a mac (Intel, running OS X v. 10.5.3) and got things up and running. (Note: this is current as of June 2008 but I strongly advise you to check the official SciPy download page before installing anything, as their information should always be up-to-date.)

  1. Pre-requisites: X11, X-code tools, c and fortran compilers, TeX, dvipng, and fftw are some basics. The first two come with Mac OS X (though not with the default installation; make sure to do a custom installation and enable these options), the compilers can be downloaded (see the installation page link above; gcc comes with the X-code tools), and the last few are available via the i-Installer.
  2. Download and install active state python version 2.5. (MacPython also works, as far as I know.) As of OS X 10.5, macs come with python v. 2.5; the binaries that I used (see below) don't work with that specific python distribution, however.
  3. Download and run the scipy superpack installer for python2.5 (note there is also a version for OS 10.4, which I haven't tested, on the same page). Numpy defines arrays and how to manipulate them and also has a number of useful routines, e.g., for generating random numbers; scipy provides additional support for other important tasks such as special functions, integration, and interpolation; matplotlib is a matlab-style plotting module. There are a few other components installed as well, including ipython, (an interactive interpreter, very nice) and readline.
  4. Find versions of ipythonrc and matplotlibrc online and modify them to suit your needs. At a minimum, be sure to set backend to TkAgg and numerix to numpy in matplotlibrc; to use LaTeX for typesetting, set text.usetex to True.
  5. Make sure the executables are in your path. In bash, this means adding something like the following line to your .bashrc:
    export PYTHONPATH='/Library/Frameworks/Python.framework/Versions/2.5/lib/python2.5/site-packages/':${PYTHONPATH}
    (modify this based on the location of your python installation). Eventually you should also add the directories with your python code to the PYTHONPATH environment variable.

That's it - your computer should now be a lean, mean, number-crunching machine. Go nuts.

Some python tips

Odds and ends that may be useful:

  • For IDL afficiandos: an IDL-numpy command comparison list (from mathesaurus). Note some of the important differences, e.g., python is row-major while IDL is column major and python indexing does not include the last element (so array[:N] returns the first N elements of the array, up to and including array[N-1]. In IDL, array[0:N] returns the first N+1 elements, up to and including array[N]).
  • Also for those coming from IDL: there is a python / numpy equivalent to the reverse_indices keyword in the histogram function. See the documentation for digitize and bincount (to be used in conjunction with the histogram function).
  • A surprising (to me, at least) "feature" of numpy arrays is that copying an array to another variable keeps the two arrays linked in memory, meaning that changing the array copy changes the original array. A detailed description of how / when arrays are copied is available at the copies and views section of the numpy tutorial, but here's an example: var1=numpy.array([1,2,3,4,5]) ; var2=var1; var2[0]=200 would result in both var1 and var2 being ([200,2,3,4,5]). For a copy that is at a distinct location in memory (a "deep copy"), use the copy method: var1=numpy.array([1,2,3,4,5]) ; var2=var1.copy() ; var2[0]=200 changes var2[0] but not var1[0]. Note that this also applies to slices.
  • Python arrays can be indexed them from the end in addition to from the beginning. Example: var1=numpy.array([1,2,3,4,5]); print var1[-2] will return 4 (the 2nd element from the end of the array). This can be pretty useful if you want to know what the last element of an array is but you don't know the length of the array.
  • The searchsorted function / method from numpy: read about it and use it, or live to regret it. Given a sorted array var1, searchsorted tells you where the first instance of another number is in that array. Example: var1=numpy.array([1,4,6,7,8,9]); print var1.searchsorted(5) will print 2 (i.e. "5" would need to be inserted at the 2nd element of the array (counting from 0) in order to preserve the array ordering). This gets to be really useful when you want to find repeated or unique indexes. For example, if we have var1=numpy.array([1, 1, 2, 3]) and var2=numpy.array([1, 3, 32, 7, -53, 26]), executing var1.searchsorted(var2, 'right') - var1.searchsorted(var2, 'left') will return array([2, 1, 0, 0, 0, 0]), i.e. for each element of var2, it will returns the number of occurrences of that element in var1. Note that var2 does not have to be sorted in this case, though var1 does. You can also use this to find the number of occurrences of each element in a specific array. var1=numpy.array([1, 1, 2, 3, 4, 5, 6, 6, 42]); var1.searchsorted(var1, 'right') - var1.searchsorted(var1, 'left') will return array([2, 2, 1, 1, 1, 1, 2, 2, 1]), i.e., var1[0] occurs twice in var1, var1[1] occurs twice, var1[2] occurs once, etc.
  • In ipython, you can run batch scripts via "run -i batch_script.py" (if you want a clean namespace for your script, drop the -i).
  • One of the most useful attributes of plotting via matplotlib is that it understands TeX commands. In order to make even mildly complex strings, put an "r" (for raw) in front of the string, such as: ylabel(r'$\rho (M_{\odot}\, \rm{kpc}^{-3})$')
  • It is easy to log a session: simply run "logstart [filename]" and all previous and future commands in the ipython session are saved into a file (either "ipython_log.py" or "filename", depending on whether or not you provided a filename when invoking logstart). Need to stop logging temporarily? No problem, just use "logoff" to stop logging temporarily and "logon" to re-start it.
